API Usage Documentation

This API usage documentation will demonstrate how the API should be used

Testing server

For now we don't have a testing server so our testing servers are staging servers. The endpoint for our staging servers is:

http://staging.housecare.tokyo/api

User add workflow

When a user wishes to perform cleanings with us, please register them with their username using

POST /users/sign_up

You will receive an API token with that that you can use to perform API calls on the user's behalf.

Note: There is no way at the moment for existing users to connect but that is on the roadmap.

Property add workflow

When a user wishes to add his properties to clean, please use this to add the property:

POST /properties/create_airbnb

There may be a few fields that require a user's input such as explanations about how to enter the room, the customer guide and throw garbage away.

In addition you need to create an estimate for the room. This is done by requesting the information required for the room via:

GET /estimates/required_fields

and post the required fields in to create the estimate:

POST /estimates

Once this is done, it is then possible to create bookings.

Booking a cleaning

When you require a cleaning simply call this API:

POST /bookings

and insert the required information start date and if possible previous and next guest numbers.
