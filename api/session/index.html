HouseCare API Documentation

POST /session

Create an api session token (Log in to HouseCare)

Parameters:

  • email - The email address of the account
  • password - The password of the account

Returns:

  • api_token - The API token of the session created

Example:

$ curl -X POST --header 'Content-Type: application/x-www-form-urlencoded' --header 'Accept: application/json' -d 'email=johnsmith%40housecare.tokyo&password=abcabcabcabc' 'https://housecare.tokyo/api/session'

{
    "api_token": "7VLnHV9Pg6S5xA9-9_a8"
}

Example error:

$ curl -X POST --header 'Content-Type: application/x-www-form-urlencoded' --header 'Accept: application/json' -d 'email=pikachu%40housecare.tokyo&password=pikapikachu' 'https://housecare.tokyo/api/session'

{
    "error": "Invalid Email or Password."
}

GET /session

Check that the api session token is valid

Parameters:

  • api_token - The API token

Returns:

  • message - Will be "Logged in" if the API token is valid for use. This member will only be returned if the HTTP status code is 200
  • error - An error message if the API token is invalid. This member will not be returned if the HTTP status code is 200

Example:

$ curl -X GET --header 'Accept: application/json' 'https://housecare.tokyo/api/session?api_token=7VLnHV9Pg6S5xA9-9_a8'

{
    "message": "Logged in"
}

Example error:

$ curl -X GET --header 'Accept: application/json' 'https://housecare.tokyo/api/session?api_token=7VLnHV9Pg6S5xA9-9_a8'

{
    "error": "Not logged in"
}

DELETE /session

Revoke an api session token (Log out of HouseCare)

Parameters:

  • api_token - The API token

Returns:

The response is empty and the HTTP status code will be 204 when the logout is successful

  • error - An error message if the API token is invalid. This member will not be returned if the HTTP status code is 204

Example error:

$ curl -X DELETE --header 'Accept: application/json' 'https://housecare.tokyo/api/session?api_token=TVLBnewe2323'

{
    "error": "Not logged in"
}
